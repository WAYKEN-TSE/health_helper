# 🏸 羽毛球饮食决策系统 (Badminton Meal App) - 用户使用手册

本手册详细说明了“羽毛球饮食决策 App”的各项功能模块、核心交互逻辑以及进阶使用技巧，帮助你（及任何使用者）快速掌握该系统。

---

## 📅 1. 核心功能：智能餐次转盘 (老虎机模块)

该模块为你提供一日三餐（早餐、午餐、晚餐）的饮食建议。它的核心是一套带有高度拟真物理“阻尼回落”感的老虎机转盘系统。

### 功能说明：
*   **一键随机抽签**：点击主页面的 `一键随机` 按钮，三餐转盘将同时启动并在 3 秒内陆续停止，向你输出基于当前库存的随机有效菜谱。
*   **单餐独立重抽**：如果对某一餐的随机结果不满意，直接点击该餐次所在的老虎机显示框，即可针对该单餐进行独立重抽。
*   **锁定心仪餐次 (锁定机制)**：老虎机右侧带有一个 🔓/🔒 锁扣图标。如果你对当前的午餐非常满意，可以点击锁扣将其锁定锁定 (🔒)。此时再点击“一键随机”，被锁定的餐次将不再变动。
*   **菜谱详情直达**：老虎机停稳后，下方会展示该菜谱的关键配料明细。点击改区域可弹出详细的烹饪步骤和宏量营养素参考指标。

---

## 🛒 2. 库存控制：高级食材管理面板

转盘能抽出什么菜，完全取决于你在【食材管理】面板中的勾选状态。该模块采用了高度直觉化的“分类治之”交互架构。

### 2.1 状态与库存联动
*   **勾选入池**：食材必须被勾选（渲染为高亮绿色），所有**依赖该食材**的预设菜谱才会被合法释放进入这顿餐次的老虎机奖池。
*   **批量操作**：每个大类（如“蛋白质”、“碳水”）右上角均附带轻量级的「全选」与「全不选」交互文字，让你能在采购日一次性补齐对应分类状态。

### 2.2 自定义食材录入 (+)
*   **极简添加**：点击对应分类名称右侧的 `+` 虚线框，即可呼出行内输入。
*   **自动归类**：在此处输入的自定义食材，不仅会自动继承当前分类标签（绝不乱序），还会立刻勾选入库并进入系统奖池环境。

### 2.3 深度删除模式 (-) 与“软删除”
*   **进入删除模式**：点击分类栏右上角的 `-` 虚线框，该分类将立刻进入红色警戒**删除模式**。此时你可以自由勾选想彻底踢出视野的食材，点击「确认」执行销毁。
*   **官方数据软删除安全阀**：为保证引擎不出错，对于带有关联菜谱的“系统内置食材”，系统会聪慧地执行“软隐藏”（仅做标记，不在界面展示）。不破坏整体菜谱的完整性。
*   **极客复活机制 (Name Sniffing)**：如果你后悔删除了某项官方预设食材（例如“鸡胸肉”），只需通过上述的 `+` 号重新手动输入“鸡胸肉”，系统在匹配同名后，会智能解冻该官方数据并原地“复活”，连带恢复关联的所有隐藏食谱。

## 📱 3. 部署与环境说明

本应用程序属于现代化 PWA (Progressive Web App) 单页应用，所有数据依托于设备的本地持久层运作。

### 离线与极客体验
当你通过 Vercel 或手机自带的「添加到主屏幕」在手机端创建快捷方式后：
*   具有防跳出、沉浸式状态栏体验。
*   支持断网运作——应用核心计算、动画引擎均跑在本地缓存中无网络调控请求，即便是在网络信号极差的羽毛球场地下室或更衣室，照样能够零延迟运行。

## 🔧 常见排错与重置

由于应用具有高强度的无缝静默状态持久化（保存你的操作习惯不断联），如果你的数据发生错乱想要**全面洗牌重来**：
1. 打开浏览器的开发者工具 (Desktop: F12)。
2. 进入 `Application` -> `Local Storage` 选项卡，清空对应 LocalHost 或域名下的键值对；或者利用清空缓存重新载入。所有预设数据与黑名单池将恢复“出厂设置”。

## 🚀 4. 开发者部署指南 (For Developers)

如果你想在自己的机器上运行、进行二次开发，或是将其公共部署以便团队/他人使用，请遵循以下基于 Vue 3 + Vite 的标准工作流：

### 4.1 环境要求
*   **Node.js**: 请确保本地已安装 Node.js (推荐 `v18.0.0` 或最新长期支持版 LTS)。
*   **Git**: 用于版本控制和同步代码。

### 4.2 本地安装与启动

1. **克隆代码库到本地**
   ```bash
   git clone https://github.com/你的账号/badminton-meal.git
   cd badminton-meal
   ```

2. **安装核心依赖项**
   ```bash
   npm install
   ```

3. **启动带热更新 (HMR) 的本地开发服务器**
   ```bash
   npm run dev
   ```
   *小贴士：控制台会输出一个 Local 地址 (如 `http://localhost:5173`) 供你在浏览器内开发。若需在手机上预览测试，在同一 Wi-Fi 下可强制输入 `npm run dev -- --host` 获取局域网 IP 并访问。*

### 4.3 终态打包与公共部署

本应用完全作为离线友好的纯前端架构设计，绝不依赖任何传统后端数据库系统。你可以将其无缝“白嫖”托管到各大免费静态网站服务中：

1. **生成高度压缩的静态资源目录**
   ```bash
   npm run build
   ```
   *执行完毕后，所有可用代码会被编译至工程下的 `dist/` 文件夹中。你需要部署的仅仅是这一个文件夹里的产物。*

2. **主流云端一键托管方案推荐**
   - **Netlify (最简命令流)**：直接在控制台敲击 `npx netlify-cli deploy --prod`，授权登录并指向 `dist` 目录，2 分钟内即上线。
   - **Vercel (最配 GitHub 行动)**：在 Vercel 官网关联你的 GitHub 仓库根目录，框架预设选择 `Vite`，每次你 `git push` 给主分支时，它都会自动化帮你走完 Build 并推送发布。
   - **GitHub Pages**：你可以通过 GitHub Actions 编写简单脚本将 `dist` 推至 `gh-pages` 分支做到站内免费分发。并且此举对于推广和 PWA 的直接挂载最为方便。
